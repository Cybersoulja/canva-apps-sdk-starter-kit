## 2024-06-13 - Add Loading States to Upload Buttons
**Learning:** For asynchronous actions that do not show an immediate UI element like an external dialog (e.g. file uploading, modifying the design canvas), using the `loading={true}` state on buttons is a critical feedback mechanism. It helps prevent multi-clicks and signals work is in progress. The `@canva/app-ui-kit` provides a `loading` prop on `<Button>` components exactly for this purpose. When applying loading states, each distinct action button should have its own tracked state to prevent all buttons on the page from showing a loading state at the same time.
**Action:** Always wrap `async` button actions in a `try...finally` block, updating a boolean `loading` state to provide responsive visual feedback. Avoid combining loading states unless the actions explicitly happen synchronously together.
